## What was painful today (gotchas to remember)

1. **Worktree Vercel link.** Deploying from a fresh worktree without `dashboard/.vercel/project.json` silently creates a NEW Vercel project named after the directory (`dashboard`) instead of linking to `tradingbot-dashboard`. Always run `npx vercel link --yes --project tradingbot-dashboard` first. (Memory saved at `feedback_vercel_worktree.md`.)

2. **Alpaca SDK lacks `getOrder()` (singular).** The TS SDK only has `getOrders()` (plural, list endpoint). Singular fetches must go through `alpacaTrade(mode, '/v2/orders/${id}')` from `dashboard/api/_lib/data-api.ts`. The `as any` cast hides this at compile time but crashes at runtime — caught only when the cron failed in production.

3. **lightweight-charts v5 API differs from v4.** Use `chart.addSeries(LineSeries, opts)` and `createSeriesMarkers(series, markers)`. The Phase 2 plan was written with v4 syntax; the implementer adapted at execution time. Worth documenting once permanently in CLAUDE.md (which I did).

4. **`trades:index:open` MUST use atomic Redis list ops** (`rpush` / `lrange` / `lrem`) — not JSON-stored array with read-modify-write. The original Phase 2 plan had a race condition that was caught in final review. Don't reintroduce it.

5. **Plan-level UX gaps** that surfaced post-deploy:
   - `/lookup` → `/order/new` click navigation wasn't wired (fixed `e4c0327`)
   - Order form header was hardcoded "loading…" string (fixed `ca4692e`)
   - No way to switch accounts on the order form (fixed `ca4692e` — added selector)
   - Canceled entry orders never updated the trade record (fixed `5188ca1`)

   Lesson for Phase 3 plan: walk through the user flow page-by-page during plan-writing, not just file-by-file. UX gaps live in the spaces between files.
