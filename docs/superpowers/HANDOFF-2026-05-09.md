## What was painful today (gotchas to remember)

1. **Plan-vs-reality drift after a rebase.** The plan was written before main shipped (a) the third paper account (`manual_paper`), (b) `Trade.modify_history`, (c) `account-utils.ts`, (d) the OrdersFilterBar, and (e) other things. The brainstorm + plan callouts at the top of the spec/plan docs flagged the deltas; each subagent was briefed with the actual on-disk shapes rather than the plan's stale assumptions. **Lesson:** when a plan is written days before execution, re-read the whole plan against `main` before dispatching the first subagent, and call out adaptations explicitly in each prompt.

2. **`requireAuth` returns `Session | null`, not `{ ok: true|false }`.** The plan invented an async-shape that doesn't match the existing handler convention. Fixed mid-flight in M2.1 — pattern is `if (!requireAuth(req, res)) return;` (matches `settings/[resource].ts`).

3. **Watchlist KV is a `string[]`, not `{symbol, added_at, note}[]`.** Plan invented richer shape; reality is simpler. Adapted M6's Watchlist page.

4. **`config.MODES` doesn't have strategy params** — they live in `strategy.py` as module constants (`STOP_PCT`, `TRAIL_TRIGGER_PCT`, `LADDERS`). The push script reads from both modules.

5. **`congress-copy/config.py` shape clashes with the top-level `config` import via Python's module cache.** The push script uses `importlib.util.spec_from_file_location` to load it under a unique name. Documented inline.

6. **`SIZING_TIERS` ends with `Decimal('Infinity')`** — JSON has no `Infinity`. The push script substitutes `1e18` as a sentinel; the dashboard-side type doc-comment notes UI should detect and render as "no cap".

7. **Calibration scatter caption directionality** — `meanDelta = user_grade - ai_grade`. With `gradeToNum` mapping higher letter → higher number, positive delta means user grades higher than AI. Plan code had this inverted (typo); caught in M6 review and fixed in commit `ce9ba89`.

8. **Calibration math must exclude `ai_grade_inherited` trades.** Otherwise the AI grade copied onto an assignment-spawned stock trade gets double-counted against the parent put's grade. Done both server-side in `/api/trades/list` (M5.4) and `/api/trades/performance` (M6 task B).

9. **Grade-cron's drain must run even on idle ticks** (no open trades to grade). M5.2 implementer caught this — if a put got assigned in tick N and the open list happened to be empty in tick N+1, the spawn would be deferred indefinitely. Removed the early-return for empty open list.

10. **Subagent-driven-development efficiency.** Tasks 1.1 → 7.x took ~2 hours of wall clock with ~30 subagent dispatches. Combining tightly-related tasks (e.g., M3.2-3.8 as one task, M5.3+5.4) saved significant overhead vs strict one-task-per-dispatch.
